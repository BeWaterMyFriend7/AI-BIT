# SkillBridge 操作手册

## 什么是 SkillBridge

SkillBridge 是一个本地 Web 管理工具，帮你在一台电脑上管理多个 AI 编码 Agent（如 Codex、OpenCode、Claude Code）的 Skill 目录。它的核心功能是：**把某个 Agent 已安装的 Skill"共享"给其他 Agent**，无需重复安装，无需复制文件。

它的原理是 Windows 的 NTFS Junction（目录链接），相当于文件系统层面的"快捷方式"，不占用额外磁盘空间。

---

## 快速开始

### 启动

```bash
cd skill-bridge
python main.py
```

打开浏览器访问 `http://127.0.0.1:17890`。

### 首次配置

1. 打开页面后，你会看到预置的 Codex 和 OpenCode 两个 Agent。
2. 如果根目录路径不对，点击右上角 **Agent 设置** → 编辑按钮修改路径。
3. 回到主页面，点击 **重新扫描**。

## 页面说明

### 1. Skill 总览（首页）

这是主页面，以 Skill 为行、Agent 为列展示状态矩阵。

- **左侧**：Skill 名称、描述
- **右侧**：每个 Agent 对应的状态图标
- **顶部工具栏**：搜索、批量操作、重新扫描、Agent 设置

每个状态图标都是可点击的按钮：

| 图标样式 | 状态 | 含义 | 点击操作 |
|----------|------|------|----------|
| 🔒 绿色锁图标 | 源 | 该 Skill 的真实文件存在于此 Agent | 不可关闭（需先迁移源） |
| ✓ 绿色勾选 | 已共享 | 已创建 Junction 链接 | 确认后取消共享 |
| ○ 灰色空心 | 未启用 | 无目录 | 创建 Junction 启用共享 |
| ● 蓝色实心 | 本地副本 | 有独立目录但非源 | 可备份后转为共享 |
| ! 黄色感叹号 | 冲突 | 同名目录但内容不同 | 需选择哪个为源 |
| ✗ 红色叉号 | 断链 | Junction 目标已不存在 | 修复或重新指定源 |
| ⊗ 灰红色 | 无效 | 缺少 SKILL.md 或无权限 | 检查目录 |

### 2. Agent 设置（/settings）

管理所有 Agent 的配置。

- **启用/停用**：复选框控制该 Agent 是否参与扫描
- **编辑**：修改名称、Skill 根目录
- **删除**：移除 Agent 配置（不会删除任何文件）
- **新增 Agent**：填写 ID、名称、Skill 根目录

> **选择目录**：点击"选择"按钮，会弹出 Windows 原生文件夹选择器，选择 Agent 的 Skill 根目录。

### 3. Skill 详情（点击 Skill 名称）

查看某个 Skill 在所有 Agent 下的详细状态：

- 各 Agent 的目录路径、条目类型（真实目录/Junction）
- 源选择/迁移操作
- 冲突详情

### 4. 操作日志（/logs）

记录所有共享启用、停用、回滚等操作：

- 时间、Action、Skill、Agent
- 操作结果（成功/失败）
- 回滚按钮（仅支持可回滚的操作）

## 常用操作

### 启用共享

1. 在主页面找到目标 Skill 行
2. 点击目标 Agent 列的灰色圆圈图标
3. 系统自动创建 Junction，图标变为绿色勾选

### 取消共享

1. 点击绿色勾选图标
2. 确认后系统仅删除 Junction，源 Skill 不受影响
3. 图标变回灰色圆圈

### 批量操作

1. 勾选多个 Skill 左侧的复选框
2. 点击顶部**批量启用**或**批量停用**
3. 选择目标 Agent，确认执行

### 处理冲突

当两个 Agent 有同名但内容不同的目录时：

1. 点击感叹号图标进入冲突详情
2. 选择其中一个作为源
3. 另一个副本会被备份后转为 Junction

### 回滚操作

1. 进入**操作日志**页面
2. 找到目标操作
3. 点击**回滚**按钮

## 常见问题

### Q: 共享后文件存放在哪里？

文件只存在源 Agent 的目录中（原始位置）。其他 Agent 通过 Junction 指向源目录，不复制文件。修改文件会在所有 Agent 中立即可见。

### Q: 取消共享会删除源文件吗？

**不会**。取消共享仅删除 Junction（目录链接），源目录和文件保持完整。

### Q: 为什么有些 Skill 显示为"无效"？

可能原因：缺少 `SKILL.md` 文件、目录无读取权限、或目录名为空/非法。建议检查该目录是否包含有效的 Skill 文件。

### Q: 配置了路径但扫描失败？

- 确认路径存在且可访问
- 检查是否有权限（文件资源管理器打开看看）
- 路径中不要包含 `~` 等 shell 缩写，使用完整绝对路径

### Q: 端口被占用？

默认端口 17890。可在 `skillbridge/config.py` 中修改 `PORT` 常量。

### Q: 如何备份数据？

数据库文件位于 `~/.skillbridge/skillbridge.db`。备份此文件即可保留所有配置和日志。
